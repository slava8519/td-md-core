# ЭТАЛОННЫЕ ДАННЫЕ (Golden Reference) — Al/Morse
## v1.0 · 2026-06-05 · для Test_0_Step и теста детерминизма (VerifyLab)

Самодостаточный золотой датасет: CI и базовая валидация **не требуют LAMMPS**. Силы проверены конечными разностями.

## Что это

Кристаллит алюминия, ГЦК, суперъячейка 3×3×2 = **72 атома** (как в тесте диссертации, Гл. 3.5), с PBC. К идеальной решётке применено сидированное случайное смещение, чтобы силы были ненулевыми и зависели от порядка суммирования (это и проверяет ядро).

## Проверенные эталонные величины

| Величина | Значение |
|----------|----------|
| Полная потенциальная энергия | **14.7286803884 эВ** |
| max\|F\| | **1.4380905392 эВ/Å** |
| Контроль конечными разностями (max\|$F_{анал}-F_{ЧР}$\|) | **2.43e-10 эВ/Å** ✓ |

> Положительная $E_{pot}$ ожидаема: $r_0=3.297$ Å (минимум Морзе) больше ГЦК-расстояния ближайших соседей (2.86 Å) — конфигурация сжата относительно минимума пары. Для теста сил это несущественно; первичный эталон — **силы**, и они верифицированы.

## Файлы

- `al_fcc_72.data` — геометрия в формате LAMMPS `read_data` (metal).
- `al_fcc_72.xyz` — та же геометрия в XYZ.
- `reference_forces.csv` — эталонные силы на атом: `id, fx, fy, fz` (эВ/Å).
- `reference_summary.json` — все параметры и контрольные суммы.
- `in.al_morse` — вход LAMMPS для независимого воспроизведения.
- `generate_reference.py` — генератор (детерминированный; пересоздаёт всё).

## Как использовать в Test_0_Step

1. Загрузить `al_fcc_72.data`, задать Морзе с параметрами выше, `R_cut=4.0`, `shift yes`, PBC.
2. Посчитать силы на нулевом шаге.
3. Сверить с `reference_forces.csv`: критерий приёмки **$\max|F_{движок}-F_{эталон}| \le 10^{-6}$ эВ/Å** (FP64). Для FP32-ядра допуск ослабляется (рекомендация ~$10^{-3}$, фиксируется в конфиге теста).

## Независимая проверка через LAMMPS (опционально)

```bash
lmp -in in.al_morse        # создаст forces.dump
# сравнить forces.dump с reference_forces.csv
```

## Воспроизвести/перегенерировать

```bash
python3 generate_reference.py   # детерминирован (seed=42), численно проверяет себя
```

## Тест детерминизма (INV-9)

Та же геометрия прогоняется в TD-движке в режиме FP64 на 1 стриме и на N стримах — координаты и скорости обязаны совпасть **побитово** (воспроизводит нулевое отклонение, диссертация Гл. 3.6, рис. 43–44).
