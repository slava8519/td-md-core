# ГЛОССАРИЙ TD-MD Core
## v1.0 · 2026-06-05 · единая терминология для разработки

Термины метода декомпозиции времени и движка. В скобках — раздел диссертации или тег `[ENG]` (инженерное расширение, в первоисточнике не валидировано).

### Метод и физика

- **ММД** — метод молекулярной динамики: интегрирование уравнений Ньютона для системы атомов.
- **Метод декомпозиции времени (TD, Time Decomposition)** — параллелизм по времени: разные временные шаги интегрирования считаются одновременно на разных узлах для разных пространственных зон (Гл. 2.1).
- **$R_{cut}$ (радиус обрезания)** — радиус потенциала взаимодействия. Атомы на расстоянии $>R_{cut}$ причинно независимы в пределах одного шага (Гл. 2.1, стр. 1807).
- **Причинный лаг** — правило: узел $P_{k+1}$ (шаг $h{+}1$) не может считать зону, пока $P_k$ (шаг $h$) её не завершил и не передал (Гл. 2.1).
- **Расчётная зона $S_i$** — слой ячеек вдоль оси декомпозиции; минимальный размер ограничен $R_{cut}$ (Гл. 2.1–2.2).
- **Буферная зона $R_{buf}$** — защита причинности на границе: $R_{buf}=v_{max}\cdot\Delta t\cdot C$, $C\ge1$ (Гл. 2.1, ур. 33). Ни один атом не должен пересечь её за шаг.
- **Ячейка** — элемент пространственной сетки; в диссертации размер выбран так, что ≤1 атом на ячейку (Гл. 3.1; для Al ≈ 2.33 Å). В движке сетка используется для пространственной сортировки → варп-кластеров.
- **Циклические граничные условия (PBC)** — замыкание модели по координате; поддержаны методом (Гл. 2.4, стр. 936–942), требуют учёта временного сдвига при замыкании кольца зон.

### Конечный автомат зон (Гл. 2.1, стр. 560–610)

- **Тип зоны** — динамическое состояние (буквы по диссертации, приведены 2026-06-11): `o` свободна · `d` данные приняты · `w` частичное взаимодействие учтено · `c` рассчитывается · `p` готова к передаче.
- **Жизненный цикл** — `o → d → w → c → p → o` (5 переходов = 5 правил смены типа). Детали — `TD_MD_Core_ZoneFSM_v1_0.md`.
- **Частичность (`w`)** — флаг: матрица сил содержит вклад лишь одного соседа (3-й закон Ньютона); зону нельзя интегрировать.

### Архитектура движка `[ENG]`

- **Узел / логическое кольцо** — единица конвейера; локально эмулируется как `cudaStream_t`, на кластере — физический GPU (Гл. 2.1; реализация — ТЗ §2.3).
- **Latency hiding** — маскировка передачи зоны вычислением; на кластере главный эффект метода (Гл. 3.4).
- **P2P masking %** — метрика: доля времени передачи P2P, перекрытая вычислением. Проявляется только multi-GPU.
- **GPU-Resident** — весь главный цикл на GPU, без `cudaDeviceSynchronize` в цикле; синхронизация шагов — через `cudaEvent_t`.
- **HAL** — лёгкий C++-слой аппаратной абстракции (вместо Kokkos); прячет вендорный API и транспорт зон.
- **SoA (Structure of Arrays)** — раскладка данных атомов по отдельным массивам координат/скоростей/сил.
- **Mixed precision** — FP64 для глобальных координат и накопления сил, FP32 для локальных смещений и расчёта сил (см. `Units`).
- **Local offsets** — координаты внутри зоны как FP32-смещение от левой границы; экономия трафика.
- **Warp-Aligned Clustering** — упаковка атомов в блоки по 32 (размер варпа) вместо списков соседей; против Thread Divergence.
- **Детерминированный режим** — FP64 everywhere + фиксированный порядок редукции → побитовая воспроизводимость (повторяет нулевое отклонение, Гл. 3.6).

### Потенциалы (ТЗ §5)

- **Морзе** — парный потенциал, эталон ядра (валидирован в диссертации для Al).
- **EAM / FS / MEAM** — многочастичные потенциалы металлов/сплавов; `[ENG]` (метод их допускает, Гл. 3.2, но в диссертации не тестировались).
- **ML-потенциалы** — нейросетевые потенциалы (Tensor Cores); `[ENG]`.
- **ReaxFF / PME / QEq** — **запрещены**: глобальная редукция на каждом шаге разрушает асинхронный конвейер TD.

### Численные методы и валидация

- **Скоростная форма алгоритма Варлета** — интегратор (Гл. 1.2, ур. 15–16).
- **Авто-шаг ($C_1$, $K_2$, $C_3$)** — $C_1$ ограничение перемещения (приоритет), $K_2$ лимит прироста T [K], $C_3$ порог смены шага (Гл. 3.3, 3.5).
- **NVE** — микроканонический ансамбль (постоянны N, V, E); единственный реализуемый в v1. NVT/NPT — в backlog.
- **VerifyLab** — подсистема кросс-валидации (мост к liblammps, golden-данные).
- **RDF** — радиальная функция распределения; статистический критерий совпадения траекторий (≥99.5%).
- **Rescue dump** — аварийное сохранение геометрии при сбое CUDA или нарушении причинности.

### DevOps `[ENG]`

- **NVTX** — маркеры NVIDIA Tools Extension для профилирования перекрытия в Nsight Systems.
- **Apptainer / Spack** — контейнеризация и пакетный менеджер для HPC-деплоя.
- **Walking skeleton** — минимальный сквозной прогон (1 зона, Морзе, NVE, FP64) до наращивания TD-сложности.
