# ЕДИНИЦЫ И КОНСТАНТЫ (Units & Conventions)
## TD-MD Core · v1.0 · 2026-06-05 · обязательно к прочтению перед любым ядром

**Проблема, которую решает документ.** Диссертация считает в эВ, Å, а.е.м., время в $10^{-14}$ с (Гл. 3.5, стр. 1207). Кросс-валидация (VerifyLab) идёт против LAMMPS. Без единой системы единиц `Test_0_Step` не сойдётся, а ошибки в множителях времени/скорости «тихие». **Решение: базовая система — LAMMPS `metal`.** Все внутренние расчёты — в metal.

---

## 1. Базовая система единиц: `metal`

| Величина | Единица | Обозначение |
|----------|---------|-------------|
| Масса | граммы/моль (= а.е.м.) | amu |
| Расстояние | ангстрем | Å |
| Время | пикосекунда | ps |
| Энергия | электронвольт | eV |
| Скорость | Å/ps | Å·ps⁻¹ |
| Сила | eV/Å | eV·Å⁻¹ |
| Температура | кельвин | K |
| Давление | бар | bar |
| Заряд | заряд электрона | e |

Выбор обоснован: эВ/Å/а.е.м. совпадают с диссертацией напрямую; отличается только время (ps vs $10^{-14}$ с), для которого ниже дан точный множитель.

---

## 2. Физические константы (значения CODATA, как в LAMMPS metal)

| Константа | Значение | Единицы |
|-----------|----------|---------|
| Постоянная Больцмана $k_B$ | `8.617333262e-5` | eV/K |
| `mvv2e` (масса·скорость² → энергия) | `1.0364269e-4` | — |
| `ftm2v` = 1/`mvv2e` (сила/масса·время → скорость) | `9648.5336` | — |
| Заряд электрона | `1.602176634e-19` | C |

**Кинетическая энергия:** $E_{kin}[\text{eV}] = \tfrac12\cdot\texttt{mvv2e}\cdot m[\text{amu}]\cdot v^2[(\text{Å/ps})^2]$.
**Ускорение для интегратора:** $a[\text{Å/ps}^2] = \texttt{ftm2v}\cdot \dfrac{F[\text{eV/Å}]}{m[\text{amu}]}$.
**Температура:** $T[\text{K}] = \dfrac{2 E_{kin}}{N_{dof}\,k_B}$, где $N_{dof}=3N-N_{constr}$ (для NVE без связей $3N-3$, если убран импульс ЦМ).

---

## 3. Перевод из единиц диссертации

Диссертационная единица времени $\tau_d = 10^{-14}$ с $= 0.01$ ps.

| Величина | Из диссертации | В metal | Множитель |
|----------|----------------|---------|-----------|
| Время / шаг $\Delta t$ | в ед. $10^{-14}$ с | ps | `× 0.01` |
| Скорость | Å·$(10^{-14}\,$с$)^{-1}$ | Å/ps | `× 100` |
| Энергия | eV | eV | `× 1` |
| Расстояние | Å | Å | `× 1` |
| Масса | a.е.м. | amu | `× 1` |

**Пример.** Шаг $\Delta t = 1.0$ в единицах диссертации $= 0.01$ ps $= 10$ фс. Коэффициент $C_1$ (макс. перемещение 0.233 Å) и $K_2$ (прирост T, K) — безразмерны/в K и переносятся как есть.

## 4. Соглашения по точности (связь с ТЗ §4)

| Данные | Тип | Обоснование |
|--------|-----|-------------|
| Глобальные координаты | FP64 (Å) | без дрейфа на длинных траекториях |
| Локальные смещения в зоне / трафик | FP32 (Å от левой границы) | экономия 50% шины; точность ~$10^{-5}$ Å при ширине зоны ≤ ~100 Å |
| Расчёт сил | FP32 (или шаблон `Real`) | скорость GPU |
| Накопление (редукция) сил | **FP64, фиксированный порядок** | детерминизм (INV-9) |
| Скорости | FP32/FP64 (конфиг) | по режиму точности |

**Правило:** ядра шаблонизированы по `template<typename Real>` (одна кодовая база, две инстанциации FP32/FP64), см. карту модулей.

---

## 5. Контроль на старте (предстартовый чек-лист)

Движок при запуске обязан напечатать активную систему единиц, значения $k_B$, `ftm2v`, и предупредить, если входные данные (из `read_data`) объявлены в иных единицах. Несовпадение единиц потенциала и геометрии — фатальная ошибка, не warning.

---

## 6. Якорь для кросс-валидации

Эталонный датасет (`reference_data/`) сгенерирован строго в metal: эВ, Å, amu, PBC. LAMMPS-воспроизведение — `pair_style morse`, `units metal`. Любое ядро сил сверяется с ним в metal без промежуточных пересчётов.
