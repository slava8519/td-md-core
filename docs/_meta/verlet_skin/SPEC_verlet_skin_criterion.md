## 1. Назначение и область

Подсистема заменяет перестройку cell-list **на каждом проходе** (3 ядра + CUB-скан на зону; силовое ядро метёт 27-ячеечную окрестность, ~142 кандидата/атом ради ~12 соседей) на **персистентный пер-зонный список Верле**, переиспользуемый `K>1` проходов, с детерминированным критерием перестройки. Подсистема ОБЯЗАНА сохранять все унаследованные инварианты (§3.1) без введения новых глобальных редукций или барьеров шага.

**В области:** критерий перестройки (L1: гибридный лаговый; L2: дрейф-корректированный), несомость на Λ-цепочке, fallback к cell-raster, материализация/переиспользование списка, валидация. **Вне области (OOS):** пер-зонные эпохи перестройки, home-zone собственность пар, компрессия индексов соседей, миграция (отложена; см. NL-INV-7).

## 2. Терминология и обозначения

| Символ | Значение |
|---|---|
| `r_cut` | радиус обрезания потенциала |
| `s` (skin) | ширина буфера списка; радиус списка = `r_cut + s` |
| `R_buf` | допустимое смещение атома за шаг: `R_buf = C_buf · v_pred · dt` (INV-4) |
| `C_buf` | множитель буфера, по умолчанию `1.5` (= диссертационный `C1`) |
| `C1` (движка) | кэп авто-шага: `v_max·dt ≤ C1·cell`, по умолчанию `≈0.1` |
| `v_pred` | прогнозная верхняя граница скорости: `v_pred = v + a·dt·lag`, `lag = n−1` (INV-4) |
| `z`, `n` | число расчётных зон / длина кольцевого контура |
| `L` | лаг Λ-цепочки = один оборот кольца ≈ `z` проходов |
| `d_i(t)` | смещение атома `i` с последней перестройки: `‖x_i(t) − x_i(t₀)‖` |
| `d_(1),d_(2)` | два наибольших `d_i` по зоне |
| `K` | фактор переиспользования списка (проходов между перестройками) |
| `t₀, t_m` | эпоха последней перестройки / последнего точного замера смещения |
| `C` | множество кандидатов списка (радиус `r_cut+s` на `t₀`) |
| `P_t` | множество физически взаимодействующих пар (`r<r_cut`) на шаге `t` |
| `Λ-цепочка` | позиционно-форвардящийся лаговый агрегат заголовков зон (несёт `dt_next`, safety bounds) |
| `Q24.40` | int64 fixed-point, порядконезависимая аккумуляция (механизм B1) |

## 3. Инварианты

### 3.1. Унаследованные (ссылка, не переопределение)

- **INV-4 (причинность).** Ни один атом не пересекает `R_buf = C_buf·v_pred·dt` за шаг. Проверяется по **фактическому** `s.v_max` каждый проход; нарушение → **HALT до коммита сил**.
- **INV-9 (побитовость).** 1-vs-z, run-to-run, в обоих режимах (fixed/mixed), через Q24.40: смена *множества кандидатов* при неизменных предикате приёма и квантовании даёт **побитово те же аккумуляторы**.
- **C-NOGLOBAL.** ЗАПРЕЩЕНО вводить новую глобальную редукцию или барьер на шаге интегрирования. `dt`/`v_max` несутся локально/позиционно по Λ-цепочке.
- **C-STATIC.** Членство зон статично (миграция отложена); страж StaleZone (дрейф ≤ `g=(w−rcut)/2`).

### 3.2. Инварианты подсистемы (NL-INV-*)

**NL-INV-1 (Суперсет — несущий).** Для каждого закоммиченного шага `t ∈ [t₀, t_next)` ОБЯЗАНО `P_t ⊆ C`. Нарушение — **физически неверный результат**, даже если все тесты детерминизма зелены. Это первичная корректностная цель подсистемы.

**NL-INV-2 (Безопасность критерия).** Решение `rebuild_now` ОБЯЗАНО срабатывать до того, как может нарушиться NL-INV-1. Достаточное условие — критерий §4, в котором заряд за проход ОБЯЗАН быть `2·R_buf` (не `2·v_pred·dt`); см. NL-INV-2a.

> **NL-INV-2a (Несущая правка заряда).** Заряд за проход ОБЯЗАН быть `charge = 2·R_buf = 2·C_buf·v_pred·dt`. Заряд `2·v_pred·dt` недосчитывает ровно в `C_buf` ⇒ пара со старта `r_cut+s−ε`, сближаясь на макс. INV-4-легальной скорости, входит внутрь `r_cut` пока `skin_consumed < s` ⇒ выпадает из списка ⇒ её сила молча `=0`. Ошибка детерминирована ⇒ 1-vs-z и run-to-run остаются ЗЕЛЁНЫМИ. Заряд `2·R_buf` подвязывает корректность под фактический HALT INV-4 (**одна оценка `R_buf` — две гарантии**).

**NL-INV-3 (Z-независимость решения).** Любой скаляр, влияющий на путь силы или на коммит (`skin_consumed`, `rebuild_now`, `verlet_active`, и L2: `d_(1),d_(2),D₀,G`), ОБЯЗАН быть бит-идентичен на всех `z`, run-to-run, в mixed. Достигается несомостью на уже-z-независимой Λ-цепочке + Q24.40 для любых сумм-редукций.

**NL-INV-4 (Единый писатель + полная реплика).** `skin_consumed`, `rebuild_now`, `verlet_active` ОБЯЗАНО вычисляться **только головой** (где `sent==0`, там же где `dt_next`) и **реплицироваться в КАЖДЫЙ исходящий заголовок прохода** (не точка-в-точку, как `dt_next`). Все `n` зон прохода ОБЯЗАНЫ почтить одно и то же решение на одном логическом шаге. (Иначе зоны перестроятся на `n−1` проходов позже → разрыв суперсета на границе.)

**NL-INV-5 (Суперсет-переиспользование).** Силовое ядро на каждом шаге ОБЯЗАНО итерировать список и **перетестировать `r²<r_cut²`** тем же предикатом приёма и тем же Q24.40-квантованием, что и rebuild-every-step. Список — целочисленные индексы (B1-прозрачен). Это даёт побитовую эквивалентность переиспользования перестройке-каждый-шаг, **пока список — суперсет** (NL-INV-1).

**NL-INV-6 (Детерминизм fallback).** Переключение Verlet↔cell-raster (`verlet_active`) ОБЯЗАНО быть детерминированной функцией z-независимых скаляров (`skin`, `R_buf`). Все зоны набора ОБЯЗАНЫ флипать синхронно. Переход `cell-raster → Verlet` ОБЯЗАН форсировать перестройку (нет валидного списка). См. §5.3.

**NL-INV-7 (Миграция).** Пока действует C-STATIC, проход миграции (когда будет введён) ОБЯЗАН форсировать перестройку. Стабильная идентичность пары (`home-zone`, `local-index`) законна **только** при статическом членстве.

## 4. Критерий перестройки (нормативно)

### 4.1. Продакшн-критерий (L1) — гибридный лаговый

Точный `d_(1),d_(2)` (два наибольших смещения с `t₀`) известны с лагом `L`, несутся на Λ-цепочке. Для `t > t_m`:

$$
\boxed{\;skin\_used(t) = d_{(1)}(t_m) + d_{(2)}(t_m) + 2\!\!\sum_{\tau=t_m}^{t-1}\! R_{buf}(\tau)\;}
\qquad R_{buf}(\tau)=C_{buf}\,v_{pred}(\tau)\,dt(\tau).
$$

Перестройка ОБЯЗАНА произойти до `skin_used(t) ≥ s`. Если `rebuild_now` вступает в силу через `L` шагов — добавить лаговый резерв `Δs_lag = 2·Σ_{τ=t}^{t+L−1} R_buf(τ)`.

- Префикс `d_(1)+d_(2)` (улучшение **И4**) — точная граница сближения по парам через две наибольшие, строго туже `2·d_max`, столь же дёшев (два порядконезависимых max на Λ-цепочке). Допустимое упрощение: `2·d_max(t_m)` (безопасно, но слабее).
- Хвост `2·Σ R_buf` покрывает движение с `t_m` (оба быстрейших атома пары — до `R_buf`/шаг на едином `v_pred`).

**Доминирование (свойство, не требование).** В не-HALT прогоне `d_(1)(t_m)+d_(2)(t_m) ≤ 2·Σ_{t₀}^{t_m−1} R_buf` ⇒ `skin_used ≤ skin_used^{cons} = 2·Σ_{t₀}^{t−1} R_buf`, строго `<` когда реальное движение ниже бюджета. Гибрид строго доминирует чисто консервативное правило `2·R_buf`.

### 4.2. Детерминизм критерия

`skin_consumed` — скалярная рекуррентность над z-независимым лаговым агрегатом; единственный писатель — голова (NL-INV-4); `skin_consumed ≥ skin` — один FP64-компар бит-идентичных значений ⇒ булев флаг бит-идентичен (NL-INV-3). `d_(1),d_(2)` — порядконезависимые max-редьюсы, несомые с лагом `L`. **Ноль новых глобальных редукций** (C-NOGLOBAL соблюдён).

### 4.3. Дрейф-коррекция (L2) — атака на ударный/высокоскоростной режим

Опциональное ужесточение для коррелированного течения, где `2·d_max` пессимистичен (соседи фронта летят вместе). Полные доказательства — `DRIFT_CORRECTION_PROOF_2026-06-13.md`.

**Однородный дрейф (Теорема 1).** `δ_i = Δx_i − D₀`, `D₀ = (1/N)Σ Δx_i`. Критерий: перестройка до `d̃_(1)+d̃_(2) ≥ s` (на остатках `d̃_i=‖δ_i‖`). Безопасен при **любом** `D₀` (граница сближения зависит лишь от разности `Δx_i−Δx_j`, дрейф сокращается тождественно).

**Аффинное поле (Теорема 2).** `δ_i = Δx_i − D₀ − G(x_i⁰−x_c)`. Критерий: перестройка до `d̃_(1)+d̃_(2) ≥ σ_min(I+G)·(r_cut+s) − r_cut`. При `G=0` вырождается в Теорему 1.

**Версии хвоста.** *Дешёвая (L2a):* хвост остаётся `2·Σ R_buf` (абсолютный) — выигрыш в префиксе, цена ~0 (вектор `D₀` на Λ-цепочке). *Полная (L2b):* хвост `2·Σ R̃_buf` по относительной скорости `ṽ` — требует правки INV-4 (HALT по `ṽ_max`); включать только если хвост — узкое место.

**Детерминизм.** `D₀` (3) / `G` (9) — Q24.40 sum-редукции (внешние произведения для `G`), бит-идентичны на `z` (NL-INV-3), несомы с лагом `L`.

**Граница применимости.** L2 чисто помогает однородному течению/сжатию. Резкий фронт (разрыв скорости) глобальным полем 1-го порядка не берётся — OOS. Включение L2 СЛЕДУЕТ гейтить по `‖G‖`/доле объяснённой дисперсии (ниже порога — выключить, аналог §5).

### 4.4. Запрещённые критерии

| Подход | Вердикт | Причина |
|---|---|---|
| Классический global `d_max` каждый шаг | **ЗАПРЕЩЕНО** | синхронный allreduce-барьер → срыв TD-конвейера, z-зависимость дерева редукции (нарушает C-NOGLOBAL) |
| Локальный `d_max` зоны / пер-зонные эпохи | **ЗАПРЕЩЕНО (L1)** | несинхронные перестройки в разных декомпозициях → разрыв 1-vs-z (нарушает NL-INV-3) |
| Home-zone собственность пары при миграции | **OOS** | смена `local-index` → ссылка на чужой атом (нарушает NL-INV-7) |

## 5. Структуры данных и алгоритм

### 5.1. Заголовок зоны

```cpp
// core::ZoneHeader (transport.hpp)
struct ZoneHeader {
    // ... существующие поля (dt_next, агрегаты Λ-цепочки) ...
    double  skin_consumed;   // L1: несомый skin-бюджет
    uint8_t rebuild_now;     // L1: флаг перестройки (реплика во ВСЕ заголовки)
    uint8_t verlet_active;   // L1: 1 = Verlet-путь, 0 = cell-raster fallback
    // L2 (дрейф): double dmax1, dmax2; double D0[3], G[9];  // несутся с лагом L
};
```

Прочее: персистентный пер-зонный Verlet-буфер (индексы, **~84 Б/атом**, **ВНЕ слот-пула**); `x_ref = x(t₀)` (24 Б/атом). Память на 1e7: +0.84 ГБ к ~1.63–2.42 ГиБ (бюджет 3.5 ГиБ). Конфиг: `skin`, `C_buf=1.5`, `C1≈0.1`, `K_on≈3.0`, `K_off≈1.5`, `verlet_default≈0`.

### 5.2. Решение головы (flush_sends, блок `sent==0`)

```cpp
// R_buf = C_buf * v_pred * dt  (посчитан при arrival 0)
const double charge = 2.0 * R_buf;             // NL-INV-2a
const double K_pred = skin / charge;           // = skin / (2·R_buf)

// NL-INV-6: гистерезис против дребезга (каждый 0→1 форсирует rebuild)
uint8_t va = verlet_active_prev;
if (va == 0 && K_pred >= cfg.K_on)  va = 1;
if (va == 1 && K_pred <  cfg.K_off) va = 0;
verlet_active = va;

if (verlet_active) {
    skin_consumed += charge;
    rebuild_now    = (skin_consumed >= skin);
    if (rebuild_now) skin_consumed = 0.0;
} else { skin_consumed = 0.0; rebuild_now = 0; }

// NL-INV-4: реплицировать skin_consumed, rebuild_now, verlet_active во ВСЕ заголовки
```

### 5.3. Перестройка / переиспользование (arrival 0, ensure_drift)

```cpp
const bool turned_on   = (hdr.verlet_active == 1) && (verlet_active_prev == 0);
const bool eff_rebuild = hdr.rebuild_now || turned_on || migration_pass;  // NL-INV-6,7

if (hdr.verlet_active) {
    if (eff_rebuild) { /* cell_count+scan+scatter + материализация Verlet-списка */ }
    else             { /* переиспользование списка (NL-INV-5) */ }
} else {
    /* Базис A: cell-raster — силовое ядро метёт 27-ячеечную окрестность; список не материализуется */
}
verlet_active_prev = hdr.verlet_active;
```

### 5.4. Машина состояний

```
                K_pred < K_off
        ┌──────────────────────────────┐
        ▼                              │
  ┌───────────┐  K_pred ≥ K_on    ┌────────────────┐  skin<skin_consumed  ┌──────────────┐
  │CELL_RASTER│ ──(force rebuild)─▶│ VERLET_REBUILD │ ───────────────────▶ │ VERLET_REUSE │
  │ (Базис A) │                   │ (материализация)│◀── rebuild_now ──────│ (переисп.)   │
  └───────────┘                   └────────────────┘  (или migration_pass) └──────────────┘
        ▲                                                                         │
        └─────────────────────── K_pred < K_off ──────────────────────────────────┘
```

### 5.5. Геометрия зоны

`make_zone_grid` halo → `rcut+skin`; ширина `w > rcut+skin` валидируется **всегда** (envelope под `skin` фиксирован, даже при `verlet_active=0`, чтобы переход `0→1` не требовал переразбиения). Адаптируется лишь *каденция* (`rebuild_now`) и *включённость* (`verlet_active`), не геометрия. Память при долгом fallback: буфер держать аллокированным (не реаллоцировать на ходу), не итерировать.

## 6. Валидация (нормативно)

> **Фундамент:** 1-vs-z и run-to-run **НЕ ловят** пропущенную пару (обе декомпозиции детерминированно промахиваются по одной паре при одинаковом шаге перестройки → зелено). NL-INV-1 ОБЯЗАН проверяться отдельным геометрическим оракулом.

### 6.1. Physical Oracle (тест-сборка)

Формально: `P_t = {(i,j): i<j, ‖x_i(t)−x_j(t)‖ < r_cut}`, `C = {(i,j): i<j, ‖x_i(t₀)−x_j(t₀)‖ ≤ r_cut+s}`. Оракул на **каждом шаге** независимо ищет все пары `r<r_cut` (cell-list «каждый-с-каждым» внутри зоны), сравнивает с парами, реально обработанными силовым ядром. Пара с `r<r_cut`, отсутствующая в списке → **abort + дамп координат/скоростей нарушителей**.

### 6.2. Обязательные CI Gates

Блокирующие коммит/слияние. ID совместимы с источниками.

**Корректность (L1):**

| Gate | Триггер | Условие прохождения |
|---|---|---|
| **GATE-01 (Oracle collision)** | high-T режим, ≥5000 шагов | ноль пропущенных Оракулом пар (NL-INV-1) |
| **GATE-02 (Force-array equivalence)** | длинный авто-шаг с близкими сближениями + тепловой спайк | траектория переиспользования **ПОБИТОВО ≡** rebuild-every-step **на уровне квантованных силовых массивов** (не координат после N шагов). Ловит зазор `C_buf` (NL-INV-2a) и лаг-рамп |
| **GATE-03 (1-vs-z identity)** | `z ∈ {1,2,4,8,16}`, fixed+mixed | бит-идентичность конечного состояния (NL-INV-3) |
| **GATE-04 (HALT-before-commit)** | нарушение бюджета шага | abort **до** коммита stale-list сил (INV-4) |
| **GATE-05 (Rebuild propagation)** | любой проход | `rebuild_now` и `verlet_active` видны **каждому** заголовку прохода (NL-INV-4) |

**Fallback (L1, И1):**

| Gate | Триггер | Условие прохождения |
|---|---|---|
| **GATE-06 (Fallback determinism)** | медленно меняющийся `v_max`, пересекающий `K*` туда-обратно | (a) `z∈{1,2,4,8,16}` флипают `verlet_active` на одном шаге; (b) число переходов ограничено (нет дребезга); (c) `0→1` форсирует rebuild; **stale-after-0→1 negative** (подавить `turned_on` → Оракул ОБЯЗАН поймать пропуск) |
| **GATE-07 (Graceful degradation)** | глубокий удар (`K→1`) | уход в `CELL_RASTER`; время шага ≤ baseline-A + ε (не «baseline + налог») |

**Производительность (L1):**

| Gate | Триггер | Условие прохождения |
|---|---|---|
| **GATE-08 (Perf/K regression)** | изменения ядра | нет роста избыточных перестроек; occupancy не падает >5% |

**Дрейф-коррекция (L2, при включении §4.3):**

| Gate | Триггер | Условие прохождения |
|---|---|---|
| **GATE-09 (D₀/G determinism)** | объёмное течение, длинный прогон | `D₀,G,σ_min` бит-идентичны на `z∈{1,2,4,8,16}` (Q24.40, не FP64) |
| **GATE-10 (Compressing-front oracle)** | не-кандидат на `r_cut+s+ε`, глобальная компрессия `σ_min<1` | критерий `σ_min(I+G)(r_cut+s)−r_cut` перестраивает **до** втягивания пары внутрь `r_cut` |
| **GATE-11 (Drift gain regression)** | прогон с дрейфом / прогон с резким фронтом | `K↑` против плоского гибрида при дрейфе; `K` НЕ ниже плоского гибрида на резком фронте |

### 6.3. Состязательные сценарии (входят в GATE-01/02/06/10)

Пара-граница (`r_cut+s−ε`, сведение на макс. INV-4-скорости); лаг-адверсарий (большой `z`, раздутый `v_pred`, узкий skin); граница-кроссинг (reorder/compact/миграция для любого локального дизайна); float-граница (`r²=r_cut²`, `skin_used==s`, `<` vs `<=`); сжимающий фронт (L2).

## 7. Производительность (информативно)

Модель шага: `T_step(s) = T_force(s) + T_rebuild/K + T_overhead`, `T_force ∝ α(s)=(1+s/r_cut)³`. Базисы: **A** — текущий движок (~142 кандидата/атом); **B** — persistent-список с чисто консервативным `2·R_buf`. Главный приз `A→B` (силовое ядро итерирует ~28 кандидатов вместо 142) **режименезависим**; точка безубыточности **`K≈2`** (материализация списка ≈ одна силовая прометка). Маргинальный выигрыш гибрида над B — режимный:

| Режим | реальный `d_max` | `K` (гибрид) | примечание |
|---|---|---|---|
| Равновесие/конденсат | насыщается ~0.1–0.2 Å | 10–20 | большой выигрыш гибрида |
| Высокая скорость/удар/газ (цель TD §3) | `≈` баллистич. бюджет | 1–3 | гибрид ~0; ценность — graceful degradation (GATE-07); L2 — частичная атака |

`K` падает с числом узлов (`v_pred` раздут на `lag=n−1`) — структурный пессимизм. `K_cons ≈ m/(2·C_buf·C1·(1+lag)) ≈ m/0.3` при `skin=m·cell`.

## 8. Фазирование и открытые вопросы

**L1 (сейчас):** persistent-список по умолчанию; критерий §4.1 (с И4) + NL-INV-2a + реплика NL-INV-4; fallback §5 (И1); Physical Oracle + GATE-01…08. **Не принимать реализацию без GATE-01 и GATE-02.**

**L2 (после L1):** дрейф-коррекция §4.3 — сначала дешёвая (L2a), полная (L2b) только если хвост — узкое место; GATE-09…11.

**OOS / открыто:** резкий фронт (разрыв скорости) — глобальное поле 1-го порядка не ловит; пер-региональный дрейф рискует 1-vs-z; пер-зонные эпохи, home-zone собственность, компрессия индексов соседей, миграция (NL-INV-7) — отдельные SPEC.
