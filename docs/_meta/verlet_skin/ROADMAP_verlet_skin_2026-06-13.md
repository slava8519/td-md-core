## 0. Что подтвердил grounding (и что изменилось против SPEC)

**Подтверждено как есть:**
- **Header-врезка тривиальна и безопасна.** 3 поля в `core::ZoneHeader` (`transport.hpp:30-38`) авто-распространяются через `cuda::GpuHeader` (`conveyor_gpu.cuh:186-190`) → host-SPSC → `MpiRingEdge` (динамический `resize`, **правок не нужно**). Поля **не входят** в int32-пакинг B5 (он на payload, не на header). Wire-дельта **+16 Б** (GpuHeader 64→80). POD, MPI_BYTE-безопасно, выравнивание без дыр.
- **Несущая правка `2·R_buf` подтверждена в коде.** `compute_R_buf = v_max·dt·C` (`buffer.hpp:33-34`), `v_pred = lf.v + lf.a·dt·lag` (`conveyor_gpu.cuh:687-688`), INV-4 по фактическому `s.v_max` с HALT в `sync_check` (`:847`). `R_buf` посчитан при arrival и **в области видимости всего прохода** — доступен в точке решения головы.
- **Идентичность i→i стабильна** (статическое членство): `upload_zone` кладёт `members[]` по порядку в слот, pack/unpack/D2D сохраняют индекс. Self-зонный сосед адресуется стабильным local-index.

**Изменения против SPEC (важно для плана):**
1. **🔴 Фича GPU-only. CPU-кольцо НЕ использует cell-list** (дисс. w-tiling, не GPU-ячейки). ⇒ Эталон для Oracle — **«GPU-Verlet ≡ GPU-rebuild-every-pass»** (оба уже ≡ CPU-кольцу). Новые поля заголовка едут по CPU-кольцу **инертно** (нужны для layout/wire-паритета, не используются). Это упрощает: критерий и список трогают только GPU-путь.
2. **🔴 Persistent-буферов НЕ существует.** `alloc_nodes` (`:472-508`) аллоцирует только слот-пул. Нужен **новый** пер-зонный буфер `verlet_buffers_[n_]` (CSR-пары + `x_ref`) **вне** переиспользуемого слот-пула (n зон покрывают все N атомов однократно). Память на 1e7: **+~1.0 ГБ** (≈84 Б/атом список + 24 Б/атом `x_ref`) к ~1.63 ГиБ (z=4/n=32 fp64) ⇒ ~2.7 ГиБ, в бюджете 3.5 ГиБ. Аллоцировать раз, в fallback не освобождать.
3. **🟡 Главный приз decoupled от сложности критерия.** Persistent-список (Базис A→B: силовое ядро итерирует ~28 кандидатов вместо 142, **режименезависимо**) даёт большой выигрыш даже с **простейшим** консервативным критерием `2·R_buf`. Гибридный префикс `d_(1)+d_(2)` лишь **дотягивает** маргинальный K. ⇒ Разделяю L1 на **L1a (простой критерий + весь приз A→B)** и **L1b (гибридный префикс)**.
4. **🔴 Ловушка, которую поймал grounding:** один агент дрейфанул к **пер-зонному** аккумулятору смещения. Это **запрещено** (SPEC §2.4: локальный `d_max` зоны → расхождение 1-vs-z). Критерий — **глобальный скаляр** на Λ-цепочке; точный префикс `d_(1)+d_(2)` — **глобальная** top-2 редукция, несомая с лагом L, не пер-зонная. Зафиксировать в ревью как анти-паттерн.
5. **🟡 Самый рискованный узел — cross-zone (halo) партнёры в reuse-ядре.** Self-зона — стабильный local-index; halo-партнёр в соседней зоне адресуется `(сторона-соседа, local-index-в-ней)`, обе со-резидентны на `launch_pairs`. Это новый kernel-контракт, не покрытый текущим `zone_pair_cells_kernel`. **Концентрация риска здесь.**
6. **🟢 min_r2/Overlap сохраняется:** любая пара под `r_min_halt` (≪ rcut < rcut+skin) — в суперсете ⇒ Overlap-HALT срабатывает корректно и на reuse-пути.

## 1. Фазы (PR-уровень, по зависимостям и риску)

### PR-0 · Каркас (header + config + no-op). Риск: низкий. ~0.5 дня
Чисто аддитивно, поведение не меняется (флаги едут, но `rebuild_now≡1` ⇒ перестройка каждый проход = текущее поведение).
- `transport.hpp:30-38`: `+double skin_consumed=0; +uint8_t rebuild_now=1; +uint8_t verlet_active=0`.
- `conveyor_gpu.cuh` оба send-сайта (`:901-914`, `:973-988`) + CPU `conveyor.hpp:448-460`: реплицировать 3 поля в **КАЖДЫЙ** заголовок (не p2p как `dt_next`).
- `config.hpp`/`config.cpp` + `ConveyorOptions` + `buffer::TimeStepCfg`: `skin`, `C_buf` (есть), `K_on≈3.0`, `K_off≈1.5`, `verlet_default=0`.
- **Gate:** существующая батарея остаётся **побитово зелёной** (поля инертны). GATE-05 (реплика во все заголовки) — новый тест.

### PR-1 · Persistent-список + простой критерий + Oracle (НЕСУЩИЙ). Риск: высокий. ~3–5 дней
Это весь главный приз. Делать как один связный PR — части не работают порознь.
- **Буферы:** `conveyor_gpu.cuh:440-460` структура `VerletZoneBuffer{int* pairs (CSR); int* counts/offsets; double* x_ref}`; alloc в `alloc_nodes` (`:~490`), free в `free_nodes` (`:~516`). Вне слот-пула, `[n_]`.
- **Геометрия:** `zone_cells.cuh:64,100-108` — `make_zone_grid` принимает `skin`; slab `range = width + 2·(rcut+skin)`, `loz -= (rcut+skin)`. `zones.hpp:46-49` — валидация `width > rcut+skin` **всегда** (envelope фиксирован, чтобы 0→1 не требовал переразбиения). `membership_guard` (`conveyor_gpu.cuh:785`): `g = 0.5·(w−rcut−skin)`.
- **Критерий (простой, L1a):** в `flush_sends` блок `sent==0` (`:907`/`:979`, там же где `dt_next`): `skin_consumed += 2·R_buf; rebuild_now = (skin_consumed≥skin); if(rebuild_now) skin_consumed=0`. **Глобальный скаляр**, без измеренного смещения. Cold start: `h==1 ⇒ eff_rebuild`; гард `R_buf<eps ⇒ K_pred=+inf`.
- **Материализация + reuse:** `ensure_drift` (`:709-734`) ветвление `if(rebuild_now){cell_count+scan+scatter + НОВОЕ store_pairs} else {reuse}`. Новый `zone_pair_verlet_kernel` (новый `zone_force_verlet.cuh`): итерирует CSR-пары, **тот же epilogue/квантование**, что `zone_pair_cells_kernel` (B1-прозрачность). **Cross-zone схема** `(сторона, local-index)` — узел риска, ревью отдельно.
- **Gates (блокирующие):** **GATE-01** Physical Oracle (брутфорс `P_t⊆C` каждый шаг, high-T, ≥5000 шагов), **GATE-02** force-array ≡ rebuild-every-pass (ловит зазор `C_buf` и лаг-рамп), **GATE-03** 1-vs-z `z∈{1,2,4,8,16}` fixed+mixed, **GATE-04** HALT-before-commit, **GATE-05** broadcast. **Не принимать PR без GATE-01 и GATE-02.**

### PR-2 · K-aware fallback (И1). Риск: средний. ~1 день
Убирает единственный downside: при `K_pred<K*` детерминированно откатываемся на cell-raster (Базис A) ⇒ худший случай = текущий движок, не «текущий + налог». Делает persistent-список безопасным **по умолчанию**.
- `flush_sends sent==0`: `K_pred=skin/(2·R_buf)`; гистерезис двумя порогами (`K_on`/`K_off`) → `verlet_active`; бюджет считать только при `verlet_active`. Деталь — `K_FALLBACK_INTEGRATION` §2.2.
- `ensure_drift`: `eff_rebuild = rebuild_now || turned_on || migration_pass`; `turned_on (0→1)` форсирует rebuild (списка ещё нет).
- **Gates:** **GATE-06** fallback-детерминизм (флип `verlet_active` на одном шаге на всех z; нет дребезга; stale-after-0→1 negative — Oracle ловит подавленный `turned_on`), **GATE-07** graceful degradation (глубокий удар → CELL_RASTER, время ≤ baseline-A + ε), **GATE-08** perf/K-регрессия в `bench_conveyor.cu:154-174` (счётчик steps-between-rebuilds, `--no-verlet`/`--K-*` флаги).

### PR-3 · Гибридный префикс `d_(1)+d_(2)` (И4/И2). Риск: средний. ~1–2 дня
Дотягивает маргинальный K в равновесии (строго доминирует `2·R_buf`). Только после того как PR-1/2 показали профиль.
- Новая **глобальная** top-2 max-редукция `‖x−x_ref‖` (порядконезависимая, Q24.40-класс), несомая на Λ-цепочке с лагом L рядом с `v_max`. `x_ref` уже хранится (PR-1).
- Критерий: `skin_used = d_(1)(t_m)+d_(2)(t_m) + 2·Σ R_buf` (SPEC §4.1) + лаговый резерв `Δs_lag`.
- **Gates:** GATE-01/02/03 остаются зелёными; **K-improvement регрессия** (K растёт против плоского `2·R_buf` в равновесии, НЕ падает в ударе).

### PR-4 · Дрейф-коррекция L2 (И3). Риск: высокий / research. Гейтить включением
Атака на ударный режим, где гибрид бессилен. Делать **только если** профиль ударного/многоузлового режима требует, и сначала дешёвую версию.
- **L2a (дешёвая):** Теорема 1 — глобальный `D₀=⟨Δx⟩` (Q24.40 sum-редукция), префикс на остатках `d̃`, хвост абсолютный. ~12 скаляров на Λ-цепочке (`DRIFT_CORRECTION_PROOF` §3.1).
- **L2b (полная):** дрейф-корректированный хвост `2·Σ R̃_buf` — **трогает INV-4** (HALT по `ṽ_max`). Только если хвост — узкое место.
- Аффинное поле (Теорема 2, `σ_min(I+G)`) — `+G` (9 скаляров). Резкий фронт — **OOS** (пер-региональный дрейф → риск 1-vs-z).
- **Gates:** GATE-09 (`D₀/G/σ_min` бит-идентичны на z), GATE-10 (сжимающий фронт), GATE-11 (drift-gain регрессия).

## 2. Зависимости и точки остановки

```
PR-0 ──▶ PR-1 ──▶ PR-2 ──▶ [профиль] ──▶ PR-3 ──▶ [профиль удара] ──▶ PR-4
каркас   приз+    fallback   STOP?         гибрид                      drift
         Oracle               (равновесный
                               выигрыш уже
                               получен)
```

- **После PR-2 главный приз (A→B) уже взят** в равновесном/конденсированном режиме, и downside снят. Это естественная точка остановки — PR-3/PR-4 имеют смысл только если профилирование покажет, что маргинальный K важен в целевом режиме.
- **Не ждать чуда в ударном/многоузловом режиме.** Там ценность — graceful degradation (PR-2), а не большой K. PR-4 — отдельная research-ветка.

## 3. Реестр рисков

| Риск | Где | Митигация |
|---|---|---|
| 🔴 Cross-zone halo-индексация в reuse-ядре | новый `zone_pair_verlet_kernel` | отдельное ревью контракта `(сторона,local-index)`; A/B против cells-пути на смешанных геометриях |
| 🔴 Молчаливый зазор `C_buf` (заряд `2·v_pred·dt`) | `flush_sends sent==0` | **GATE-02** force-array≡; код-ревью именно этой строки; константа из `o_.ts.C_buf` |
| 🔴 Пер-зонный аккумулятор (анти-паттерн) | соблазн в `ensure_drift` | критерий ОБЯЗАН быть глобальным скаляром; ревью-чеклист |
| 🟡 Память +1 ГБ на 1e7 | `alloc_nodes` | mem-probe в bench; буфер не освобождать в fallback |
| 🟡 MPI: голова на другом ранге | `MpiRingEdge` | broadcast пересекает boundary-edge (header едет по MPI) — GATE-05 на np≥2 |
| 🟡 Cold start / div-by-zero | pass 1, `K_pred` | `h==1⇒rebuild`; `R_buf<eps⇒+inf`; `verlet_default=0` |
| 🟢 CPU-кольцо без cells | — | поля инертны на CPU; эталон Oracle = GPU-rebuild-every-pass |

## 4. Непереуступаемо (повтор из SPEC)

- Заряд **`2·R_buf`**, не `2·v_pred·dt` (NL-INV-2a).
- `rebuild_now`/`skin_consumed`/`verlet_active` — **broadcast во все заголовки** (NL-INV-4).
- **Physical Oracle + GATE-01/02** — без них PR-1 не принимать (1-vs-z НЕ ловит пропуск пары).
- Критерий — **глобальный** скаляр на Λ-цепочке (не пер-зонный).

## 5. Решения за тобой (до старта PR-1)

1. **Значение `skin`** — фиксированное (напр. 1.0 Å ≈ 0.39σ) или конфиг? Влияет на K и память (halo). Рекоменд.: конфиг, дефолт ~1 ячейка.
2. **`verlet_default`** — список по умолчанию ON (с PR-2 fallback это безопасно) или OFF (opt-in за флагом)? Рекоменд.: после PR-2 — ON.
3. **Веха.** Это single-GPU перф-оптимизация, ортогональна M6 (физика EAM). Ставить как параллельный перф-трек «post-M5a» (как порт cell-list), не блокируя M6. Номер вехи — за тобой.
4. **Делать ли PR-4 (drift)** сейчас или отложить до измеренного потолка гибрида? Рекоменд.: отложить.
