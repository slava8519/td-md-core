## 0. Вердикт (TL;DR)

Эксперименты диссертации 2007 в их буквальном виде сегодня **не приёмочны**, но оба
актуализируются без потери трассируемости:

- **§3.5 (LJ-масштабирование 1–4 млн атомов на кластере)** → современная реплика на **одной**
  RTX 5080: LJ-система **10⁶–10⁷ атомов** — это ровно «представительный single-GPU LJ-прогон 2025»
  (LAMMPS-KOKKOS гоняет 16 млн LJ на H100 как эталонную single-GPU нагрузку; RBMD упирается в
  память на ~10 млн на A100). Наш потолок ~10⁷ на 15.5 ГиБ совпадает с этим фронтиром.
- **§3.6 («греем 72-атомную Al-наночастицу до плавления»)** → **оставить только как
  корректностно-конвейерную реплику** (побитовое 1-vs-z, свободные границы), но **снять с неё
  ярлык «измерение плавления»**: 72 атома не имеют резкого плавления (наименьший Al-кластер с
  выраженным переходом — Al₂₈⁺), плавление занижено на сотни K (Гиббс–Томсон), а одно-фазный нагрев
  завышает T_пл на ~18–19 % из-за перегрева. Современный рецензент это забракует как термодинамику.
- **Физику плавления** перенести на **двухфазный coexistence** (а не нагрев) и на **EAM-Al** (M6),
  не на парный Морзе/LJ — парные потенциалы металлическое плавление систематически врут.
- **Объективная приёмка корректности** строится на **внешних эталонах**, которых у диссертации не
  было: NIST SRSW (LJ energy/virial по замороженным конфигурациям + EOS), кросс-сверка с LAMMPS,
  тест Мерца–Ширтса (флуктуации энергии ∝ Δt²).

выполнимо. **N=72 (golden) — это correctness-фикстура, а не перф-точка** (зонд M2.5 показал то же).

## 2. Угол 2 — стандарты валидации (внешние эталоны)

**NIST SRSW (Standard Reference Simulation Website), LJ — обновлён 2026-04-08, актуален.**
- Замороженные конфигурации с эталонными **U_pair\***, **W_pair\*** (вириал), **U_LRC\*** для
  *детерминированной* проверки энергии/сил/вириала кода (напр. Config 1: U_pair\*=−4.3515·10³,
  W_pair\*=−5.6867·10², U_LRC\*=−1.9849·10² при r_c\*=3.0). **Три схемы обрезки** (r_c\*=3.0+LRC,
  4.0+LRC, 3.0+force-shift) — надо точно совпасть по cutoff.
- EOS-данные T\*=0.60…1.50 (шаг 0.05), методы NVE-MD/NVT-MD/NVT-MC/GCMC-TMMC (кросс-метод).
- Критическая точка LJ: **T_c\*=1.3120(7), ρ_c\*=0.316(1), p_c\*=0.1279(6)** — фальсифицируемый репер.
- *Все в приведённых LJ-единицах* — конвертировать из metal. (NIST SRSW, primary, 2012/обн.2026.)

**Нормы дрейфа NVE (OpenMM User Guide 8.x, primary).** Убиквитин, 0.5 фс: single **4.3·10⁻⁴**,
mixed **2.3·10⁻⁵**, double **1.1·10⁻⁷** kT/(нс·dof). В mixed/double дрейф «диффузный»
(random-walk), систематический пренебрежим — *именно так отличают численный шум от ошибки
интегрирования*. (Совпадает с цифрами, заложенными в `MIXED_PRECISION_BESTPRACTICES_2026-06-11.md`.)

**GROMACS (primary, 2026).** Единого числового порога дрейфа **нет** — настраивают
`verlet-buffer-tolerance` (дефолт 0.005 кДж/моль/пс); leap-frog достаточен для production,
velocity-Verlet точнее для обратимого сопряжения. *Вывод: абсолютный эВ-порог — не индустриальная
практика; меряют в kT/(нс·dof) либо через Δt-скейлинг.*

**Тест Мерца–Ширтса (J. Chem. Theory Comput. 2018, primary) — золото для нашего NVE-критерия.**
Для симплектического интегратора **флуктуации полной энергии ∝ Δt²**: отношение флуктуаций при
двух шагах = отношению квадратов шагов — фальсифицируемый тест. Порог систематической ошибки **>3σ**.
Открытая библиотека `physical-validation` (GROMACS/LAMMPS/HOOMD; в GROMACS с 2018). Демо-протокол:
**LJ 1000 частиц (Ar), ρ\*=0.8, бокс 3.6039 нм, шаги 4 фс→1/8 фс**.

**Кросс-движковая сверка (Vanommeslaeghe-класс, 5 движков, 2016, primary).** GROMACS/AMBER/LAMMPS/
DESMOND/CHARMM: энергии совпадают **<0.1 %** относительной при согласованных cutoff/PME (по
компонентам до 1 части на 10⁷). **Координатная точность доминирует**: ~0.75–1 цифра энергии теряется
на цифру координаты ⇒ сравнения детерминизма требуют **FP64-координат** (= наш инвариант). Вердикт
индустрии — «достаточно для практики», **не побитово** (наша внутренняя цель «1-vs-z побитово» —
*другая* и более сильная).

## 3. Угол 3 — что валидно на наших потенциалах

**Морзе-Al (Girifalco–Weizer 1959, Phys. Rev. 114(3):687; наши параметры).** Подгонян к **энергии
испарения, постоянной решётки, сжимаемости** — *не к плавлению/термике*. NIST/OpenKIM
(MO_140175748626, high-cutoff, DOI 10.25950/45d9848f): **E_coh^fcc = −2.9234 эВ/атом, a_fcc = 4.033 Å**
— готовый эталон статики. Энергии связи **почти вырождены** по фазам (fcc −2.9234, hcp −2.9233,
dhcp −2.9233, bcc −2.9092) ⇒ парный потенциал плохо различает плотноупакованные фазы (известный
минус vs EAM, рецензент отметит). **Температуры плавления NIST не даёт** — плавление не
калиброванная наблюдаемая для Морзе-Al. *(Подтверждает напрямую переданный вывод: Морзе годен для
корректности сил/энергии, не для физики плавления.)*

**Парные потенциалы и плавление металлов — систематически врут (2 независимых primary).**
Морзе-Fe coexistence: **2325 K vs эксп. 1811 K**. LJ-частица Al не плавится даже при 1200 K;
нужны многочастичные (glue, Streitz–Mintmire), Sutton–Chen тоже не справляется.

**LJ как эталон.** Тройная область ρ\*≈0.84, T\*≈0.72; критическая точка (см. §2). Демо
Мерца–Ширтса — LJ Ar ρ\*=0.8. LJ-плавление через coexistence (см. §5) воспроизводимо точно.

**EAM-Al (M6).** Для реальной физики плавления — таблицы в NIST Interatomic Potentials Repository
(Mishin 1999 и др.); EAM-наночастицы дают размерную зависимость (см. §4).

## 4. Угол 4 — размерные эффекты и плавление нанокластеров

- **Al-наночастицы (EAM, 2007/2023, primary):** T_пл растёт с размером — 473 K при 2 нм (256
  атомов) → 720–827 K при 2–4 нм → булк **937 K только при ~8 нм**. Ниже 8 нм плавление сильно
  занижено.
- **Порог «как булк»:** модели сходятся к эксперименту лишь при **n ≥ 1000 атомов** (для Au; <6000
  для Pb); ниже — сильная размерная зависимость. (primary, 2021)
- **Малые Al-кластеры (эксперимент по теплоёмкости, Phys. Rev. B 73, 125407, 2006; primary):**
  N=31–62 плавятся при **400–650 K**, **немонотонно**, магическое число **N=55** — пик, при N=56
  резкий спад; наименьший Al-кластер с выраженным переходом — **Al₂₈⁺** (пик ~600 K). ⇒
  **72 атома не имеют резкого, булк-подобного плавления.**
- **Гистерезис** плавление–кристаллизация ΔT=193–272 K ⇒ одиночное определение T_пл ненадёжно.
- **PBC-перегрев:** одно-фазное булк-плавление перелетает термодинамическое на **~18 %** (Lutsko
  поправка 0.75–0.85). (primary, 2007)
- **Общая размерная чувствительность** материал/механизм-зависима: для перехода нужно ≥ нескольких
  сотен атомов (суперионики, 2022, primary) — единого порога N нет.

**Вывод:** диссертационная 72-атомная Al-частица сегодня — *иллюстрация конвейера*, не измерение
плавления. Для физики плавления N ≥ 1000 (нанокластер) или булк с поправкой на перегрев.

## 5. Угол 5 — методология приёмочного эксперимента

**One-phase «нагрев до плавления» — не приёмочный протокол (2 независимых primary).** Перегрев
завышает T_пл на **19 %**: LJ-Ar дефект-free кристалл «плавится» при 99.6 K vs эксп. 83.81 K;
Fe — 2750–3250 K (нагрев) vs 2325 K (coexistence). **Именно этим методом мерил §3.6 диссертации.**

**Двухфазный coexistence — современный стандарт, и он работает в NVE (важно — мы NVE-only v1).**
`interface/NVE` для LJ-Ar даёт **82.8 K** (эксп. 83.81). Протокол: уравновесить твёрдо-жидкую
границу в NVE, варьировать размер бокса (плотность), записывать средние T и P, плавление — там, где
фит P–T пересекает P=1 атм. Референс: **LJ 2048 атомов, 1 фс, LAMMPS**. Для простых одноатомных
(парных) систем все методы согласуются, так что coexistence достаточен. (primary, 2012)

## 6. Рекомендуемая валидационная лестница (флагман + опора), под RTX 5080

> Привязка к вехам; VRAM/wall-clock — оценки по якорям §1; критерии — фальсифицируемые.

**🚩 ФЛАГМАН (визитка проекта) — «современная реплика §3.5»: корректность + сохранение на большой LJ.**
- *Система:* LJ, **10⁶ (CI/демо) и 10⁷ (флагман)** атомов, PBC, NVE, ρ\*≈0.8, T\* в жидкой области.
- *Что доказывает:* (а) уход от O(N²) и насыщение GPU на серьёзном N (M3); (б) сохранение энергии
  на индустриальном уровне; (в) объективную корректность против внешнего эталона.
- *Эталоны:* **NIST SRSW** — U_pair\*/W_pair\*/U_LRC\* по замороженным конфигурациям (детерминированная
  сверка сил/энергии/вириала, точное совпадение cutoff) **+** EOS T\*-скан; **тест Мерца–Ширтса**
  (флуктуации ∝ Δt², библиотека `physical-validation`); дрейф в **kT/(нс·dof)** с реперами OpenMM
  (mixed ~2·10⁻⁵, fp64 ~10⁻⁷).
- *VRAM/время:* 10⁷ ≈ ~3 ГиБ, ~30–60 мин на 10⁵ шагов; 10⁶ ≈ мин.
- *Веха-владелец:* M3 (тайминг/корректность) → M4 (на GPU, с перф) → **M5b** (масштабирование, как §3.5).

**Опора A — корректность Морзе-Al (НЕ плавление).**
- Сверка сил/энергии vs `LAMMPS pair_style morse`; статика vs NIST/OpenKIM (**E_coh=−2.9234 эВ/атом,
  a=4.033 Å**). Веха: уже частично (golden), расширить в M3/M6.

**Опора B — историческая реплика §3.6 (корректность конвейера, не термодинамика).**
- Al-72, Морзе, свободные границы, авто-шаг (C1=10), ≥25 900 шагов, **побитовое 1-vs-z** (методика
  Гл. 3.6). Покрывает путь «кольцо без PBC». **Ярлык в отчёте: «historical determinism replica, NOT
  a melting measurement».** Веха: M3.5/M4.

**Опора C — физика плавления (M6, на EAM + coexistence).**
- LJ-плавление через `interface/NVE` coexistence (2048+ атомов) → сверка с табличным LJ;
  затем **EAM-Al** (NIST IPR) coexistence → булк ~933 K с поправкой на перегрев. Протокол приёмки —
  энергия + дрейф импульса на наборе систем (образец HOOMD). НЕ Морзе/LJ для металлического плавления.

## 7. Чего НЕ делать

- ❌ **One-phase «нагрев до плавления» как измерение T_пл** — перегрев +19 %, два независимых
  источника. Только coexistence.
- ❌ **Плавление на Морзе-Al или LJ** — парные потенциалы металлическое плавление врут (Морзе-Fe
  2325 vs 1811 K; LJ-Al не плавится при 1200 K). EAM — для физики.
- ❌ **72-атомная частица как прокси булка** — нет резкого перехода ниже ~Al₂₈, занижение на сотни K,
  немонотонность, гистерезис ΔT до 272 K. Для булк-физики N ≥ 1000, лучше ≥8 нm для Al.
- ❌ **Заявлять побитовое совпадение с LAMMPS/другими движками** — индустриальная планка <0.1 %
  энергии при согласованных параметрах, *не* побитово (координатная точность доминирует). Наше
  побитовое «1-vs-z» — внутренняя цель детерминизма, её и заявлять, не межсофтовую идентичность.
- ❌ **Мерить перф на N=72/малых** — недогруз GPU (≈1 варп), ~1.4× над CPU; визитка должна быть 10⁶–10⁷.

## 8. Диссертация 2007 vs план 2026

| | Диссертация 2007 | План 2026 (TD-MD Core) |
|---|---|---|
| Масштаб (scaling) | LJ 1 и 4 млн, кластер до 32 узлов | LJ **10⁶–10⁷ на одной RTX 5080** (= single-GPU фронтир 2025), затем кластер M5b |
| Железо | кластер тех лет | RTX 5080 (sm_120), потолок ~10⁷/карта по памяти |
| «Плавление» | нагрев Al-72 до плавления (one-phase) | coexistence на LJ + **EAM**-Al (M6); Al-72 — только реплика детерминизма |
| Корректность | сверка 1-vs-z (побитово) | то же **+ внешние эталоны**: NIST SRSW, LAMMPS, Мерц–Ширтс Δt² |

| Эталон | внутренний (1 vs z) | внутренний **+ независимый внешний** (воспроизводимость как у движков 2016/2018) |
